# Project Summary
SOUL PELLEGRINI TECH is a premium institutional website for a strategic technology consulting firm specializing in digital transformation, artificial intelligence, governance, and cybersecurity. The platform targets CEOs, directors, and the healthcare sector, providing a modern interface that emphasizes services, showcases case studies, and enhances client interaction.

# Project Module Description
The project consists of several functional modules:
- **Home**: Introduction and overview of services.
- **About**: Company mission, vision, values, and a timeline focused on strategic IT consulting and digital transformation for the healthcare sector.
- **Services**: Detailed descriptions of offered services.
- **Advisory**: Information on advisory programs for executives, including participation in programs across various sectors.
- **Cases**: Showcase of successful projects, now with anonymized testimonials.
- **Contact**: Contact form and company information.
- **WhatsApp Button**: A floating button for direct communication, featuring animation and tooltip for user interaction.

# File Description Inventory
- **README.md**: Project overview and setup instructions.
- **.npmrc**: Configuration file to enforce legacy peer dependencies.
- **index.html**: Main HTML file for the application.
- **package.json**: Project dependencies and scripts.
- **package-lock.json**: Automatically generated file that locks the versions of dependencies.
- **src/App.tsx**: Main application component that defines routes and includes the WhatsApp button.
- **src/pages**: Contains all page components for the website.
- **src/components**: Contains reusable UI components, including the WhatsApp button and the footer.
- **src/hooks**: Custom hooks for managing state and logic.
- **src/lib**: Utility functions and API handling; `api.ts` is now a placeholder.
- **public/**: Static assets like images and icons.
- **vercel.json**: Configuration for Vercel deployment, specifying the root directory and build commands.

# Usage
To set up the project:
1. Install dependencies: `npm install --legacy-peer-deps`
2. Build the project: `npm run build`
3. Run the development server: `npm run dev`

### Deploying to Vercel
1. Ensure that the `.npmrc` file is present in both the project root and `app/frontend/` to manage dependencies.
2. Set the **Root Directory** to `app/frontend` in the Vercel Dashboard under Settings в†’ General.
3. Push changes to GitHub to trigger redeployment.
